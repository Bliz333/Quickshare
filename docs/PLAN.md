# QuickShare 改进规划（分支：feature/hardening-plan）

目标：补足多租户隔离、安全合规、文件夹闭环、管理员后台与易支付接入。执行顺序从安全→功能闭环→运营能力。

## 原则
- 先安全后功能：先消除越权/泄漏，再补功能。
- 环境无密钥：敏感配置全部走环境变量/本地 profile，不入库。
- 阶段收敛：每阶段可独立上线，有可验证的交付物与回滚点。

## 阶段与交付物

### 阶段 0：基线与配置清理
- 移除仓库内明文密钥（DB/Redis/JWT/SMTP/recaptcha/上传路径），新增 `application-local.yml` 示例，引导环境变量注入。
- 增加 `.env.example`（如需）与启动文档，限制 CORS 允许域。
- 交付物：安全配置文档 + 清理后的配置文件。

### 阶段 1：安全与鉴权强化（最高优先）
- 引入 Spring Security 过滤链，统一 Bearer JWT 校验，删除 `userId=1` 回退。
- User 增加 `role` 字段（默认 `USER`），JWT 携带角色；预留 `ADMIN`。
- API 访问控制：文件/文件夹/分享均强制 `user_id=currentUser` + `deleted=0`；分享访问只通过分享码+提取码。
- 全局异常与统一错误返回；限流/频率控制（验证码、分享创建）。
- 文件上传校验：类型/大小/无扩展处理；流正确关闭；物理存储路径隔离。
- 交付物：安全测试用例（鉴权、越权、上传校验）+ 更新后的接口文档。

### 阶段 2：文件夹与文件闭环
- `/api/files` 支持 `folderId` 查询且过滤逻辑删除；上传接口接受 `folderId`。
- `/api/folders/{id}/rename`、删除/递归删除、重命名冲突处理；列表同时返回文件夹和文件。
- 物理文件删除与 DB 删除对齐；ShareLink 状态/过期/次数校验健壮化。
- 前端 `netdisk.js` 与后端接口对齐，保证目录浏览/重命名/删除可用。
- 交付物：接口契约更新 + 手工/自动化用例覆盖目录操作。

### 阶段 3：管理员体系与后台
- 角色与权限：`ADMIN` 角色校验（接口 `@PreAuthorize`）。
- 管理接口：用户列表/禁用、文件列表与强制删除、分享失效、系统概览（上传量、存储占用）。
- 管理前端：独立入口（静态或小型前端）支持管理员登录与基础操作；审计日志记录管理员动作。
- 交付物：管理员 UI 原型 + 后台接口文档 + 审计日志落库策略。

### 阶段 4：易支付接入
- 数据模型：订单表（用户、金额、渠道、状态、商户单号、第三方流水、回调时间、附加信息）。
- 服务接口：创建订单（返回易支付跳转/二维码）、查询订单、易支付回调验签与状态更新。
- 业务绑定：成功支付后触发权益（如存储扩容、会员、下载券）；防重复通知与幂等处理。
- 安全：签名密钥走环境变量；回调接口限流与 IP 白名单（如支持）。
- 交付物：支付流程文档 + 回调验签测试 + 对账/审计日志。

### 阶段 5：运维与质量
- Flyway/Liquibase 建表与演进脚本；数据种子。
- 日志与监控：关键操作日志、上传/下载/分享指标。
- 自动化测试：单测（工具/服务），集成测试（鉴权、上传/目录、分享、支付回调的伪造用例）。
- 交付物：CI 脚本（如可用）、覆盖率报告、告警/日志配置说明。

## 里程碑建议
- M1（阶段 1 完成）：安全基线可上线，阻断跨用户访问。
- M2（阶段 2 完成）：网盘目录功能可用且一致。
- M3（阶段 3 完成）：管理员可运营/审计。
- M4（阶段 4 完成）：支付闭环打通。
- M5（阶段 5 完成）：可持续运维与质量保障。
