## 一、项目概述（听众版）

QuickShare 是一套**私有化部署的文件分享与个人网盘平台**。你可以把它理解成"自己搭的百度网盘 + 自己的文件分享站"，所有数据都在你自己的服务器上，完全由你掌控。

它提供三个核心能力：

| 能力 | 通俗解释 |
|------|----------|
| 文件分享 | 上传文件后生成一个链接，别人打开链接输入提取码就能下载，可以设过期时间和下载次数上限 |
| 个人网盘 | 登录后有一个自己的文件管理页面，能建文件夹、批量上传、拖拽移动文件、在线预览文档 |
| 跨设备快传（QuickDrop） | 同一个账号在手机和电脑浏览器上同时打开，文件可以直接从一台设备"飞"到另一台设备，不需要数据线，也不需要第三方 App |

除此之外，平台还内置了**套餐与支付功能**、**管理员后台**（可以控制注册开关、上传限制、邮件配置等），并且支持**中英文双语界面和深色模式**。

## 二、完整演讲稿

### 2.1 开场

大家好，感谢各位抽出时间。

今天我想介绍一个叫 QuickShare 的项目。先不谈技术，我先问一个问题：大家平时怎么把一个大文件发给同事或者朋友？

（停顿）

最常见的几种做法：微信传，但有 200MB 限制；用百度网盘，但速度被限制，而且文件存在别人那里；用邮件附件，容量更小。如果你是企业用户，还有数据安全合规的顾虑，文件不能存在第三方云盘上。

QuickShare 就是来解决这些问题的。

### 2.2 痛点

我总结一下日常文件分享的几个核心痛点：

**第一，工具碎片化。** 大文件用网盘，小文件用聊天工具，临时分享用邮件，每个场景换一个工具。体验割裂，文件散落在各处。

**第二，隐私不可控。** 用公共网盘，文件存在别人的服务器上。你对数据的存储位置、访问权限、留存周期都没有实质控制力。对于企业用户，这直接关系到数据合规。

**第三，跨设备传文件太麻烦。** 手机上收到一份文档，想传到电脑上编辑，你得先传到某个云盘，再到电脑上下载。或者找根数据线。折腾一圈，文件还没到手上。

**第四，缺乏统一管理。** 分享出去的链接没有过期机制，下载次数没有上限，谁下过、什么时候下的，一概不知。

### 2.3 方案

QuickShare 的思路很简单：**你自己搭一套文件服务，把数据放在自己可控的环境里。** 文件的访问权、留存策略、存储位置，全部由你决定；如果有需要，也可以接入 S3 兼容对象存储来承载文件。

部署方式也比较直接：准备好服务器后，可以通过 Docker Compose 快速把整套服务拉起来。不需要安装客户端，不需要下载 App，打开浏览器就行。

它把文件分享、个人网盘、跨设备快传三件事整合在一个平台里，不用再在不同工具之间来回切换。

### 2.4 核心功能详解

下面我逐个介绍 QuickShare 的核心功能。每个功能我都会用一个生活化的场景来说明。

#### 功能一：文件分享

**场景：** 你做了一份方案 PDF，需要发给客户。你不方便用微信，也不希望文件在公共网盘上长期留存。

**用法：** 上传文件，系统生成一个分享链接和一个提取码（类似百度网盘的提取码）。你可以设置链接的过期时间（比如 24 小时后自动失效）和下载次数上限（比如最多下载 5 次）。到了限制条件，链接自动失效，别人打不开。

同时，平台支持 Office 文档和 PDF 在线预览。接收方打开链接后，不用下载就能直接在浏览器里看文件内容。

#### 功能二：个人网盘

**场景：** 你日常积累了大量工作文件，需要一个统一的地方管理和查找。

**用法：** 登录后进入"我的网盘"页面，界面和常见的文件管理器类似。你可以创建多级文件夹、批量选择文件、用拖拽的方式把文件移动到不同的文件夹。上传时如果发现同样的文件已经存在，系统会自动去重，不重复占存储空间。

网盘页面还能看到当前的存储用量和套餐信息，用量一目了然。

#### 功能三：跨设备快传 QuickDrop

**场景：** 你手机浏览器上有一张照片或者一个文档，想快速传到电脑上。

**用法：** 手机和电脑都用同一个账号登录 QuickShare。打开首页的快传功能，系统自动发现你的另一台设备。选中文件，直接发送。传输过程会优先走"浏览器对浏览器"的直连通道，速度快、不经过服务器中转。如果网络条件不允许直连，会自动回退到服务器中转，保证文件一定能送达。

这个功能还有另一种用法：公开取件。你可以生成一个取件链接，发给任何人。对方打开链接就能接收文件，不需要注册账号，也不需要登录。

#### 功能四：套餐与支付

**场景：** 你想把 QuickShare 作为服务提供给其他人使用，需要收费。

**用法：** 管理员在后台配置套餐（比如"基础版 10GB"、"专业版 100GB"）和支付方式。用户在定价页面选择套餐，跳转支付，完成后自动获得对应存储空间。用户可以查看订单历史和当前套餐状态。

#### 功能五：管理后台

**场景：** 你是管理员，需要控制平台运行策略，但不想改代码、不想重启服务。

**用法：** 通过一个隐藏路径访问管理后台（路径不在公开页面暴露）。在这里你可以做这些事：

- 开关注册、控制上传大小上限、设置是否允许匿名上传
- 配置邮件服务器和邮件模板，发公告或验证邮件
- 管理套餐、查看订单、调整配额
- 查看存储状态：磁盘用了多少、剩余多少、风险等级如何
- 控制注册时的验证码方式

这些设置全部是运行时生效，改完即用，不需要重启服务。

### 2.5 价值

总结 QuickShare 带来的核心价值：

**数据自主。** 文件存在你自己的服务器上，你知道文件在哪、谁能访问、什么时候删除。这是公共网盘给不了的。

**体验统一。** 文件分享、网盘管理、跨设备传输，一个平台全搞定。不用在不同工具之间来回切换。

**部署简单。** Docker 一键启动，不需要安装客户端，浏览器就是入口。用户的学习成本几乎为零。

**灵活可控。** 管理员可以不写代码、不重启服务，在后台调整所有运行策略。从注册规则到邮件配置到支付渠道，运行时即改即生效。

**开源免费。** QuickShare 采用 MIT 开源协议。代码公开透明，你可以自由使用、修改和二次开发，没有许可费用。

### 2.6 差异化

和市面上的产品相比，QuickShare 有几个明显的差异点：

**与百度网盘等公共云盘比：** QuickShare 是私有化部署的，数据在你自己的服务器上。公共云盘的文件存储在第三方，你对数据没有实质控制权。QuickShare 的体验和能力边界更多取决于你自己的网络、硬件和部署方式。

**与 NAS 产品（如群晖）比：** NAS 需要购买专用硬件，配置门槛较高。QuickShare 只需要一台普通服务器或者云主机，通过 Docker Compose 就能把整套服务拉起来。用户端不需要安装任何软件，打开浏览器就行。

**与 WeTransfer 等临时分享服务比：** WeTransfer 只做临时文件分享，没有个人网盘功能，没有跨设备传输，也没有管理后台。QuickShare 把分享、网盘、快传、管理四件事整合在一起。

**QuickDrop 的独特之处：** 跨设备传输走的是浏览器对浏览器的直连通道（技术术语叫 WebRTC）。这意味着文件在传输过程中可以不经过服务器，速度更快，隐私更有保障。如果直连条件不满足，会自动降级到服务器中转，让传输更稳。

### 2.7 收尾

最后我想说，QuickShare 解决的核心问题不是"某个功能比别人强"，而是**把文件分享、网盘、跨设备传输、后台管理整合成一个你完全可控的私有平台**。

如果你在意数据隐私、想要统一的文件管理体验、或者需要一个可以自行定制的文件服务，QuickShare 是一个值得考虑的选择。

项目完全开源，MIT 协议，欢迎在 GitHub 上查看代码和文档。

谢谢大家。
