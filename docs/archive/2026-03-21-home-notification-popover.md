# 2026-03-21 首页通知中心改为右上角图标 + 弹出层

## 本轮目标

- 解决首页通知中心长期占用首屏空间的问题
- 保留通知能力，但把默认形态改成“右上角铃铛入口”
- 只有检测到比上次更新的通知时，才自动弹出通知中心

## 本轮改动

- 首页通知入口改为右上角铃铛图标：
  - `src/main/resources/static/index.html`
- 通知中心从正文常驻面板改为固定弹出层：
  - 点击铃铛打开
  - 点击关闭按钮、点击外部区域或按 `Esc` 关闭
- 新增本地“最近已查看通知”标记：
  - 匿名用户与登录用户分开记录
  - 首次进入页面只建立基线，不会把历史通知误判成“新通知”
  - 后续检测到更新时会自动弹出一次
- 铃铛按钮补上未读数量徽标：
  - 按“全站通知 + 个人通知”合并计数
  - 关闭通知中心时再把当前批次标记为已查看
- 通知中心补上前端本地已读能力：
  - 支持单条“标为已读”
  - 支持“全部已读”
  - 继续按匿名用户 / 登录用户分别保存本地已读状态
  - 兼容上一版只记录“最近已查看通知”的本地数据
- 通知列表补上显示节奏收口：
  - 默认先展示前 4 条
  - 支持“查看更多”分批展开剩余通知
  - 长正文默认折叠，点击后可展开/收起
- 移动端右上角按钮组改为更紧凑的胶囊式布局：
  - 减少首屏边角拥挤感
  - 保留通知、套餐、主题、语言四个高频入口
- 保留现有“全部通知 / 我的通知”切换逻辑
- 同步更新首页通知中心 Playwright 用例：
  - 匿名用户默认折叠，点击铃铛后展开
  - 登录用户存在更新通知时自动弹出
  - 新通知自动弹出时会显示未读数量徽标
  - 单条已读后数量会递减
  - 全部已读后数量和按钮会同步收口
  - 默认只展示前 4 条，点击“查看更多”后继续展开
  - 长正文支持展开/收起
  - 移动端宽度下右上角按钮组仍保持可点击和不拥挤

## 验证

- `./scripts/check-js.sh`
- `docker compose up --build -d app`
- `npx playwright test tests/e2e/home-notifications.spec.js`

## 结果

- 首页默认不再被通知中心大面板占用
- 通知中心仍然可快速访问，且“有新通知才主动弹出”的行为已经落地
- 右上角铃铛现在能直接表达“这次来了几条新通知”，而不只是一个红点
- 用户现在可以在通知中心里逐条消化通知，不需要每次都等到关闭弹层才统一清空未读
- 通知中心现在不会在通知较多或正文很长时一下子把弹层撑满，阅读节奏更稳定
- 移动端顶部入口区已经更紧凑，不会因为通知入口加入后显得松散或顶边过挤
- 这次交互调整已经形成页面级自动化回归，不再只靠手工点击确认
